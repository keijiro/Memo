# テキストエディタ

## NeoVim の限界

基本的には NeoVim で全てを賄おうというつもりでいるが、一つだけどうしても解決できない問題がある。それは日本語 Markdown の編集だ。

Markdown は英語のわかち書きを前提としている部分がある。文の途中に改行・スペース・タブが入るのが前提で、それを使って行の頭を合わせるなど行い、raw text で覗いてもある程度の見た目が担保されるという仕組みだ。

しかし日本語ではこれが破綻する。わかち書きを行わないので、余計な改行やスペースを挿入すると、レンダリング後のテキストに余計なスペースが挟まってしまうのだ。

この問題を解決するには「改行やスペースを挟まなくても見た目を整えてくれる Markdown テキストエディタ」を使用する必要がある。

NeoVim は便利だが日本語 Markdown の編集には使えない、という結論に至った。

## Typora

https://typora.io

Markdown を編集できる軽量なテキストエディタとして Typora を導入することにした。日本語の使用も問題無く、今のところは支障無く使えている。
